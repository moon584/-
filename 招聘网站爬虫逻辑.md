4.数据库job表填入规则：

| 列名            | 数据类型         | 中文说明          | 约束（含中文解释）                                           | 取值域                            | 备注                                                         |
| --------------- | ---------------- | ----------------- | ------------------------------------------------------------ | --------------------------------- | ------------------------------------------------------------ |
| id              | VARCHAR(16)      | 职位唯一标识ID    | 主键，不能为空                                               | 如 C001J00001                     | 公司id + J + 编号                                            |
| company_id      | VARCHAR(16)      | 所属公司ID        | 非空；外键，引用 company(id)；禁止删除被引用公司；公司ID更新时级联更新 | 已存在的公司ID                    | 表示该职位属于哪家公司                                       |
| category_id     | VARCHAR(16)      | 所属分类ID        | 非空；外键，引用 category(id)；禁止删除被引用分类；分类ID更新时级联更新 | 已存在的分类ID                    | 表示该职位所属岗位分类                                       |
| job_url         | VARCHAR(512)     | 职位详情页原始URL | 非空；唯一约束                                               | 合法URL地址                       | 同一职位链接不可重复                                         |
| title           | VARCHAR(200)     | 职位名称          | 非空                                                         | 200字符以内                       | 如 Java后端开发工程师                                        |
| salary          | VARCHAR(20)      | 薪资范围          | 可空，默认 '面议'                                            | 如 10k-20k、25k-45k、面议         | 无薪资默认 面议                                              |
| job_type        | TINYINT UNSIGNED | 招聘类型          | 可以为空；检查约束 job_type IN (0,1)                         | 0=社会招聘，1=校园招聘            | 数字存储，高效                                               |
| education       | VARCHAR(20)      | 学历要求          | 可空                                                         | 大专、本科、硕士、博士及以上等    | 用于筛选学历门槛                                             |
| publish_time    | DATETIME         | 职位官方发布时间  | 可空                                                         | 合法日期时间                      | 来源于招聘官网发布时间                                       |
| location        | VARCHAR(50)      | 工作地点          | 可空                                                         | 城市/地区名称，如北京、上海、深圳 | 可用于地域筛选                                               |
| description     | TEXT             | 职位职责描述      | 可空                                                         | 文本描述                          | 存储岗位职责内容                                             |
| requirement     | TEXT             | 职位任职要求      | 可空                                                         | 文本描述                          | 存储任职条件、技能要求                                       |
| crawl_status    | TINYINT UNSIGNED | 爬取状态          | 非空，默认 0；检查约束 crawl_status IN (0,1)                 | 0=失败，1=成功                    | 标记本次职位抓取是否成功                                     |
| crawled_at      | DATETIME         | 爬取时间/入库时间 | 非空，默认当前时间                                           | 合法日期时间                      | 数据进入系统时间                                             |
| created_at      | DATETIME         | 记录创建时间      | 非空，默认当前时间                                           | 合法日期时间                      | 记录首次创建时间                                             |
| bonus           | TEXT             | 加分项            | 可空                                                         | 文本描述                          | 存储优先资格、加分条件等内容，如“有ACM/ICPC奖项”“熟悉CUDA编程”等 |
| work_experience | VARCHAR(100)     | 工作经验要求      | 可空                                                         | 100字符以内                       | 存储年限或描述，如“一年以上工作经验”“3-5年相关经验”          |

6.其他要求：

- 每次抓取**增量更新**，只更改变化和crawled_at，crawl_status

- 设置合理的延迟（如 0.5~1 秒/请求），避免给服务器造成压力。

- 增设异常处理与重试

- **记录爬取日志**：打印成功/失败数量，便于监控。
